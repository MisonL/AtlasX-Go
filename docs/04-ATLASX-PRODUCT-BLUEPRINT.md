# AtlasX 产品蓝图

## 产品定位

`AtlasX` 是一个面向 Intel x64 macOS 的 `Atlas-like` 桌面浏览器产品。

它不是官方 Atlas 的补丁版，而是一个新生产品：

- 目标是尽量复原官方产品的能力面
- 不与官方闭源宿主壳耦合
- 用 Go 主导系统控制面
- 用 Chromium 承载浏览器数据面

## 设计原则

1. 先把浏览器产品能力做实，再谈 AI 外挂
2. 先建立可运行最小闭环，再切到自管 runtime
3. Go 只做它擅长的控制面与状态面，不强行承担浏览器渲染职责
4. 所有高耦合原生能力后移，先保主链可用

## 产品能力层

### Layer A: 浏览器基础层

- 标签页
- 地址栏
- 导航
- 历史
- 书签
- 下载
- 多窗口
- DevTools
- 导入

### Layer B: ChatGPT 融合层

- 侧边栏问答
- 页内总结
- 选区提问
- 上下文提取
- 项目、文件、连接器入口

### Layer C: 智能浏览层

- Agent 模式
- Browser memories
- 页面建议
- 标签自动整理
- 上下文推荐

### Layer D: 平台与治理层

- Profile
- 设置
- 默认浏览器
- 权限
- 更新
- 日志与诊断
- 企业策略

## 不做的事情

- 不试图二进制翻译官方 `arm64` 宿主壳
- 不承诺完整复刻官方服务端策略
- 不在第一阶段自研浏览器内核

## 最低完成态定义

一个合格的 `AtlasX` 最低版本至少要满足：

- x64 Mac 上独立运行
- 有独立浏览器窗口和 profile
- 能在当前页发起侧边栏问答
- 能读历史、书签、下载
- 能进行基础 Chrome/Safari 导入
- 有统一设置与诊断入口
