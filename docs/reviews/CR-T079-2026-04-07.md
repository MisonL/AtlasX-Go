- 日期: 2026-04-07
- 任务: `T079 补齐按标签页聚合的 Browser memories 入口`
- 结论: 已完成。`atlasctl tabs memories` 与 `GET /v1/tabs/memories` 已落地，CLI 与 HTTP 共用 `memory.FindRelevantSnippets` 主链，按当前标签页上下文返回相关 memory snippets，保持只读，不写 memory。

  - `atlasctl tabs memories <target-id>`
- 新增 HTTP:
  - `GET /v1/tabs/memories?id=<target-id>`
- 复用主链:
  - `tabs capture`
  - `memory.FindRelevantSnippets`
- 新增测试:
  - `atlasx/cmd/atlasctl/tabs_memories_test.go`
  - `atlasx/internal/daemon/tab_memories_test.go`

## 审查结论

- 已确认入口保持只读:
  - 仅抓取当前页上下文并执行 memory retrieval
  - 未调用 `memory.AppendPageCapture`、`memory.AppendQATurn` 或任何写入动作
- 已确认 CLI 与 HTTP 共用同一套检索逻辑:
  - 都调用 `memory.FindRelevantSnippets`
- 已确认失败显式暴露:
  - `capture` 失败时 CLI 打印 `capture_error`
  - HTTP 返回结构化 `capture_error` 与 `error`
- 已确认空结果显式返回:
  - 无 memory 时返回 `returned=0` 与 `snippets=[]`
