# CR-T065

- 任务: T065 收紧 mirror 与 chrome import 的来源目录边界
- 日期: 2026-04-07
- 结论: 通过

## 变更范围

- 新增共享路径校验模块 `internal/sourcepaths`
- `atlasctl mirror-scan` 与 `atlasd /v1/mirror/scan` 只允许 `AtlasX profiles root` 或系统 `Chrome profiles root`
- `atlasctl import-chrome` 与 `atlasd /v1/import/chrome` 只允许系统 `Chrome profiles root`
- 越界目录现在显式返回错误，不再允许任意本地目录作为数据源

2. `cd atlasx && go test ./cmd/atlasctl ./internal/daemon`
   - 退出码: 0
   - 摘要: CLI 与 HTTP 对越界目录显式拒绝，允许目录仍通过。

3. `cd atlasx && go test ./...`
   - 退出码: 0
   - 摘要: 全仓回归通过，未破坏既有 mirror/import/sidebar/tabs 主链。

## 审查结论

- 本轮关闭了一个高风险本地文件读取面：HTTP 调用方不再能把任意目录当作 Chrome profile 或 mirror 来源。
- CLI 与 HTTP 现在共用同一套路径规则，避免控制面和命令行出现双重真相。
- 当前仍未给控制面增加认证鉴权；本轮处理的是“即使能访问控制面，也不能任意指定本地目录”这一层防线。
