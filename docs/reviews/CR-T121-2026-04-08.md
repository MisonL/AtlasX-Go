## 实现事实

- `tabs` 主链新增：
  - `OpenDevToolsPanelWindowIntoWindow(sourceWindowID, panel, targetWindowID)`
  - 复用 `resolveWindowPair` 做源/目标窗口存在性与差异性校验
  - 对源窗口每个 page target 逐个复用 `OpenDevToolsPanelInWindow`
  - 返回 `source_window_id/panel/target_window_id/returned/opened_targets`
- CLI 新增：
  - `tabs open-devtools-panel-window-into-window <source-window-id> <panel> <target-window-id>`
  - 输出窗口级批量结果与逐项 opened target
- HTTP 新增：
  - `POST /v1/tabs/open-devtools-panel-window-into-window`
  - 请求体: `source_window_id`、`panel`、`target_window_id`
  - 校验窗口 ID 正整数与 `panel` 必填，错误路径显式失败

## 审查结论

- 新能力没有引入第二套 DevTools panel URL 打开逻辑，窗口级批量动作完全复用既有 `OpenDevToolsPanelInWindow` 主链
