# CR-T087

- 日期: 2026-04-08
- 任务: `T087 补齐浏览器窗口激活入口`
- 结论: 已完成。`atlasctl tabs activate-window` 与 `POST /v1/tabs/activate-window` 已落地，CLI 与 HTTP 共用 `tabs.ActivateWindow` 主链，先复用 `windows` 分组结果定位目标窗口，再复用既有 `activate(target)` 主链完成窗口聚焦，返回结构化激活结果。

## 变更事实

- 新增 `tabs.WindowActivateResult`
- 新增 `tabs.ActivateWindow(...)`
- 新增 CLI:
  - `atlasctl tabs activate-window <window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/activate-window`
- 新增测试:
  - `atlasx/internal/tabs/window_activate_test.go`
  - `atlasx/cmd/atlasctl/tabs_activate_window_test.go`
  - `atlasx/internal/daemon/tab_activate_window_test.go`
  - `atlasx/internal/daemon/tab_activate_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有两条主链:
  - 先走 `Windows()` 定位 `window_id`
  - 再走 `Activate(targetID)` 聚焦该窗口中的第一个 page target
- 已确认 CLI 与 HTTP 共用同一套 activate window 主链:
  - 都调用 `tabs.ActivateWindow(...)`
- 已确认失败显式暴露:
  - `window_id` 不存在时显式返回错误
  - `activate(target)` 失败时显式返回错误
