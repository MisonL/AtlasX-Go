# CR-T084

- 日期: 2026-04-08
- 任务: `T084 补齐浏览器窗口位置尺寸控制入口`
- 结论: 已完成。`atlasctl tabs set-window-bounds` 与 `POST /v1/tabs/window-bounds` 已落地，CLI 与 HTTP 共用 `tabs.SetWindowBounds` 主链，通过 browser websocket 调用 `Browser.setWindowBounds` 对指定窗口应用显式 `left/top/width/height`，并通过 `Browser.getWindowBounds` 回读结构化窗口 bounds。

## 变更事实

- 新增 `tabs.SetWindowBounds(...)`
- 新增 CLI:
  - `atlasctl tabs set-window-bounds <window-id> <left> <top> <width> <height>`
- 新增 HTTP:
  - `POST /v1/tabs/window-bounds`
- 新增测试:
  - `atlasx/internal/tabs/window_bounds_test.go`
  - `atlasx/cmd/atlasctl/tabs_window_bounds_test.go`
  - `atlasx/internal/daemon/tab_window_bounds_test.go`
  - `atlasx/internal/daemon/tab_window_bounds_contract_test.go`

## 审查结论

- 已确认动作只修改指定 `window_id` 的 `left/top/width/height`:
  - 主链调用 `Browser.setWindowBounds`，只传 `windowId` 和位置尺寸字段，不混入 `windowState`
- 已确认 CLI 与 HTTP 共用同一套窗口 bounds 主链:
  - 都调用 `tabs.SetWindowBounds(...)`
- 已确认失败显式暴露:
  - 非法 `window_id` 显式返回错误
  - 非法 `width/height` 显式返回错误
  - browser websocket 缺失或 bounds 设置失败显式返回错误
