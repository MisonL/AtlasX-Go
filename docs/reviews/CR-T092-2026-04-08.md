# CR-T092

- 日期: 2026-04-08
- 任务: `T092 补齐单标签跨窗口迁移入口`
- 结论: 已完成。`atlasctl tabs move-to-window` 与 `POST /v1/tabs/move-to-window` 已落地，CLI 与 HTTP 共用 `tabs.MoveToWindow` 主链，先复用 `Windows()` 定位源标签所属窗口和目标窗口，再复用 `open-in-window` 与 `close(target)` 主链把单个 page target 迁入目标窗口，返回结构化迁移结果。

## 变更事实

- 新增 `tabs.WindowMoveResult`
- 新增 `tabs.MoveToWindow(...)`
- 新增 CLI:
  - `atlasctl tabs move-to-window <tab-id> <target-window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/move-to-window`
- 新增测试:
  - `atlasx/internal/tabs/window_move_test.go`
  - `atlasx/cmd/atlasctl/tabs_move_to_window_test.go`
  - `atlasx/internal/daemon/tab_move_to_window_test.go`
  - `atlasx/internal/daemon/tab_move_to_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有三条主链:
  - 先走 `Windows()` 定位源标签所属窗口与目标窗口
  - 再走 `OpenInWindow(...)` 在目标窗口打开标签
  - 最后走 `Close(targetID)` 关闭源标签
- 已确认 CLI 与 HTTP 共用同一套 move to window 主链:
  - 都调用 `tabs.MoveToWindow(...)`
- 已确认失败显式暴露:
  - 指定 page target 不存在时显式返回错误
  - 目标窗口不存在时显式返回错误
  - 源窗口与目标窗口相同时显式返回错误
  - open 或 close 任一步失败时显式返回错误
