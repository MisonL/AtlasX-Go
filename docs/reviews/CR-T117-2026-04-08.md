## 实现事实

- `agentplan` 新增链式执行模型：
  - 错误类型：`ErrStepSequenceRequired`、`ErrStepBatchExceeded`
  - 结果结构：`BatchExecutionResult`
  - 执行入口：`ExecuteBatch(...)`
- `ExecuteBatch` 约束：
  - 必须 `confirm=true`
  - 必须显式提供 `stepIDs` 且不能为空
  - 请求步骤数超过 `max_steps` 时显式失败，并返回结构化失败结果（包含 `requested/executed/stopped/failure/max_steps`）
  - 按用户给定顺序逐步执行，复用既有单步 `Execute(...)` 主链
  - 部分失败时返回 `failed_step_id`、已执行结果集合与回滚摘要，不伪造未执行步骤结果
- CLI 入口 `tabs agent-execute`：
  - 支持 `one or more steps`
  - 新增 `--max-steps`（默认 `1`）
  - 单步保持原输出结构
  - 多步输出批量摘要和逐步结果
- HTTP 入口 `POST /v1/tabs/agent-execute`：
  - 支持 `step_id`（单步）与 `step_ids`（多步）两种请求
  - `step_id` 与 `step_ids` 互斥
  - 多步返回 `BatchExecutionResult` JSON
  - 错误路径返回对应状态码并附结构化批量失败结果
