# CR-T099

- 日期: 2026-04-08
- 任务: `T099 补齐按指定窗口建议组整理到指定现有窗口入口`
- 结论: 已完成。`atlasctl tabs organize-window-into-window` 与 `POST /v1/tabs/organize-window-into-window` 已落地，CLI 与 HTTP 共用 `tabgroups.ApplyWindowToWindow` 主链，先基于既有窗口分组定位指定源窗口，再仅对该窗口中的 page targets 生成建议组，并按组顺序逐个复用 `applyGroupToWindow` 主链把建议组整理到指定现有目标窗口，未命中建议组的标签保留在源窗口。

## 变更事实

- 新增 `tabgroups.ApplyWindowToWindow(...)`
- 新增窗口内定向整理结果:
  - `ApplyWindowIntoWindowResult`
- 新增 CLI:
  - `atlasctl tabs organize-window-into-window <source-window-id> <target-window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/organize-window-into-window`
- 新增测试:

## 审查结论

- 已确认入口只复用现有两层主链:
  - 先走 `Windows()` 定位指定源窗口中的 page targets
  - 再对该窗口 targets 走 `tabgroups.Suggest(...)`，并逐组复用 `applyGroupToWindow(...)`
- 已确认 CLI 与 HTTP 共用同一套窗口内定向整理主链:
  - 都调用 `tabgroups.ApplyWindowToWindow(...)`
- 已确认失败显式暴露:
  - `source_window_id` 或 `target_window_id` 非法时显式返回错误
  - 源窗口或目标窗口不存在时显式返回错误
  - `source_window_id` 与 `target_window_id` 相同时显式返回错误
  - 任一建议组整理失败时显式返回错误
  - 窗口内未命中建议组时显式返回空 `groups`
