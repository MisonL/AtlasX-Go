# CR-T091

- 日期: 2026-04-08
- 任务: `T091 补齐窗口合并入口`
- 结论: 已完成。`atlasctl tabs merge-window` 与 `POST /v1/tabs/merge-window` 已落地，CLI 与 HTTP 共用 `tabs.MergeWindow` 主链，先复用 `Windows()` 读取源/目标窗口，再复用 `open-in-window` 与 `close(target)` 主链把源窗口标签迁入目标窗口，返回结构化合并结果。

## 变更事实

- 新增 `tabs.WindowMergeTarget`
- 新增 `tabs.WindowMergeResult`
- 新增 `tabs.MergeWindow(...)`
- 新增 CLI:
  - `atlasctl tabs merge-window <source-window-id> <target-window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/merge-window`
- 新增测试:
  - `atlasx/internal/tabs/window_merge_test.go`
  - `atlasx/cmd/atlasctl/tabs_merge_window_test.go`
  - `atlasx/internal/daemon/tab_merge_window_test.go`
  - `atlasx/internal/daemon/tab_merge_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有三条主链:
  - 先走 `Windows()` 读取源/目标窗口
  - 再走 `OpenInWindow(...)` 在目标窗口打开标签
  - 最后走 `Close(targetID)` 关闭源窗口中的旧标签
- 已确认 CLI 与 HTTP 共用同一套 merge window 主链:
  - 都调用 `tabs.MergeWindow(...)`
- 已确认失败显式暴露:
  - source/target 相同时显式返回错误
  - 源窗口或目标窗口不存在时显式返回错误
  - 任一标签迁移失败时显式返回错误
