## 变更范围

- 为 install 失败路径补 archive/part/extract 残留清理
- 明确 rollback 触发条件：仅在安装前已有 managed runtime 备份时进入 `rollback -> rolled_back`
- 保留 rollback 后的 `last_error`，同时恢复旧 manifest 与 bundle
- 补无旧 runtime 失败清理、已有旧 runtime 回滚恢复、rollback 错误保留测试

## 审查结论

- rollback 只在 install 已备份旧 runtime 时触发；无旧 runtime 的失败保持 `failed`，避免伪造 rollback。
- install 失败后 `archive_path`、`.part` 与解包临时目录会被清理，旧 runtime 不会被误删。
- rollback 完成后 install plan phase 为 `rolled_back`，`last_error` 仍保留，恢复目标与失败证据可同时观测。
