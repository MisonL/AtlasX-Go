- 日期: 2026-04-08
- 任务: `T100 补齐按指定窗口查看整理建议入口`
- 结论: 已完成。`atlasctl tabs organize-window` 与 `GET /v1/tabs/organize-window` 已落地，CLI 与 HTTP 共用 `tabgroups.SuggestWindow` 主链，先基于既有窗口分组定位指定窗口，再仅对该窗口中的 page targets 生成结构化建议组，作为窗口级只读整理观测入口。

## 变更事实

- 新增 `tabgroups.SuggestWindow(...)`
- 新增窗口级只读建议结果:
  - `SuggestWindowResult`
- 新增 CLI:
  - `atlasctl tabs organize-window <window-id>`
- 新增 HTTP:
  - `GET /v1/tabs/organize-window?window_id=<id>`
- 新增测试:
  - `atlasx/internal/tabgroups/apply_test.go`
  - `atlasx/cmd/atlasctl/tabs_organize_window_test.go`
  - `atlasx/internal/daemon/tab_organize_window_test.go`
  - `atlasx/internal/daemon/tab_organize_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有两层只读主链:
  - 先走 `Windows()` 定位指定窗口中的 page targets
  - 再对该窗口 targets 走 `tabgroups.Suggest(...)`
- 已确认 CLI 与 HTTP 共用同一套窗口级整理建议主链:
  - 都调用 `tabgroups.SuggestWindow(...)`
- 已确认失败显式暴露:
  - `window_id` 缺失或非法时显式返回错误
  - `window_id` 不存在时显式返回错误
  - 窗口内无建议组时显式返回空 `groups`
