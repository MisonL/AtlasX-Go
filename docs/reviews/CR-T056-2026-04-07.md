# CR-T056

- 任务: T056 实现基于 memory 的轻量问答增强
- 日期: 2026-04-07
- 结论: 通过

## 变更范围

- 新增 `internal/memory` 检索器，基于 `tab_id`、`url`、`title` 和问题 token 计算相关性，只从本地 memory 事件中提取片段
- 为 memory 检索补充测试，覆盖排序、空态退化和长答案片段截断
- `sidebar` 新增 `AskWithMemory`，在不改变响应契约的前提下把相关 memory 片段并入 prompt
- provider prompt 引入显式预算控制：基础页面上下文优先，只有在 token 预算允许时才追加 `Relevant Memory` 段
- `/v1/sidebar/ask` 在调用 provider 前执行 memory 检索；memory 文件不存在时显式退化为当前页面上下文；memory 读取错误时显式返回 500

## 审查结论

- 增强逻辑只依赖本地 `events.jsonl`，没有引入向量库、外部索引或第二套状态真相。
- 预算策略是显式的：基础 prompt 先构造，memory 片段按顺序尝试加入，超预算即停止追加，而不是静默截断页面主上下文。
- 失败语义保持 Debug-First：没有 memory 文件时正常退化；memory 文件损坏或读取失败时 ask 直接报错，不会伪装成“已增强成功”。
