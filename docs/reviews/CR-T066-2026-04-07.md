# CR-T066

- 任务: T066 收紧 browser-data open 的 URL scheme 边界
- 日期: 2026-04-07
- 结论: 通过

## 变更范围

- 新增共享 URL 校验模块 `internal/openurl`
- `browserdata` 的 history、downloads、bookmarks 打开解析现在只允许 `http` 和 `https`
- `tabs` 的打开与导航入口共用同一套 URL 规则
- `atlasd` 的 `/v1/tabs/open` 现在会在执行前显式拒绝危险 scheme

## 验证记录

1. `cd atlasx && go test ./internal/browserdata`
   - 退出码: 0
   - 摘要: history、downloads、bookmarks 对 `javascript:`、`file://`、`devtools://` 的拒绝行为通过。

2. `cd atlasx && go test ./internal/daemon`
   - 退出码: 0
   - 摘要: `/v1/history/open`、`/v1/downloads/open`、`/v1/bookmarks/open` 对危险 scheme 显式失败；`/v1/tabs/open` 入口同样受限。

3. `cd atlasx && go test ./internal/tabs`
   - 退出码: 0
   - 摘要: `tabs.Client.Open` 与 `tabs.Client.Navigate` 对危险 scheme 显式拒绝。

4. `cd atlasx && go test ./...`
   - 退出码: 0
   - 摘要: 全仓回归通过，未破坏既有 daemon、browserdata、tabs、sidebar、runtime 主链。

## 审查结论

- 本轮关闭了 browser-data 到受管浏览器之间的危险 scheme 透传路径，避免把 `file://`、`javascript:`、`chrome://`、`devtools://` 直接交给浏览器执行。
- 校验已同时落在 browserdata、daemon 和 tabs 共享边界，避免只修 HTTP 层或只修客户端导致双重真相。
- 当前控制面仍然没有鉴权；本轮处理的是“即使请求能到达打开链路，也只能打开受限的 Web URL”这一层防线。
