## 变更范围

- 新增第二个 provider adapter：`openrouter`
- `sidebar ask` 增加请求级 `provider_id` 选择，未传时继续走 default provider
- 非法 `provider_id` 在抓 tab context 前显式返回 `400`
- sidebar provider 选择统一收敛到 `resolveProvider` / `Validate`，避免 ask 路径和状态面分叉
- 补充 sidebar 与 daemon 测试，覆盖 default provider、request override、openrouter 路径和错误 provider id

## 审查结论

- 现在 sidebar 至少存在两条可执行 adapter 路径：`openai` / `openai-compatible` 与 `openrouter`。
- provider 选择先走请求级 `provider_id`，缺省时回落到 default provider，没有引入静默 fallback。
- 非法 provider id 在控制面即被拒绝，不会误打到 tabs capture 或上游 provider 请求链。
