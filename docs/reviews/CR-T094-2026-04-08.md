# CR-T094

- 日期: 2026-04-08
- 任务: `T094 补齐按建议组整理到新窗口入口`
- 结论: 已完成。`atlasctl tabs organize-group-to-window` 与 `POST /v1/tabs/organize-group-to-window` 已落地，CLI 与 HTTP 共用 `tabgroups.ApplyToNewWindow` 主链，先复用既有 `tabgroups.Suggest` 定位指定建议组，再复用 `move-to-new-window` 与 `move-to-window` 主链把该组标签整理到同一新窗口，返回结构化整理结果。

## 审查结论

- 已确认入口只复用现有三条主链:
  - 先走 `tabgroups.Suggest(...)` 定位指定建议组
  - 再走 `MoveToNewWindow(...)` 拆出组内首个标签并定位新窗口
  - 最后走 `MoveToWindow(...)` 把剩余组内标签迁入该窗口
- 已确认 CLI 与 HTTP 共用同一套 organize apply 主链:
  - 都调用 `tabgroups.ApplyToNewWindow(...)`
- 已确认失败显式暴露:
  - `group_id` 不存在时显式返回错误
  - 组内标签不足两项时显式返回错误
  - 任一标签迁移失败时显式返回错误
