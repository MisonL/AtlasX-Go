## 实现事实

- `defaultbrowser` 主链新增：
  - `SetBundleID(bundleID)`
  - 先复用既有 LaunchServices 导出主链读取当前 `LSHandlers`
  - 仅重写 `http/https` 两个 scheme 的 handler 为 `RoleAll=<bundle_id>`
  - 保留其他 scheme handlers 不变
  - 通过 `plutil` 将 JSON 转回 plist，再用 `defaults import` 写回
  - 写后再次走 `ReadStatus()` 回读核验，确保 `http/https` 都指向目标 bundle id
  - `bundle_id` 为空白时显式失败
- CLI 新增：
  - `atlasctl default-browser set <bundle-id>`
  - 返回与 `status` 相同的结构化状态
- HTTP 新增：
  - `POST /v1/default-browser/set`
  - 请求体: `bundle_id`
  - 空 `bundle_id` 在 HTTP 层直接返回 `400`

- 设置动作只作用于 `http/https` 的 LaunchServices handler，没有扩散到其他 scheme
- CLI 与 HTTP 共用同一套 `defaultbrowser.SetBundleID` 主链，没有出现第二套写路径
