# CR-T080

- 日期: 2026-04-07
- 任务: `T080 补齐标签页设备模拟控制入口`
- 结论: 已完成。`atlasctl tabs emulate-device` 与 `POST /v1/tabs/emulate-device` 已落地，CLI 与 HTTP 共用 `tabs.EmulateDevice` 主链，支持固定设备预设和 `off` 显式清除，保持单标签页作用域，不引入第二套状态面。

- 新增 `tabs` 设备预设与结果结构:
  - `DevicePreset`
  - `DeviceEmulationResult`
- 新增 `tabs.EmulateDevice(...)`
- 新增 CLI:
  - `atlasctl tabs emulate-device <target-id> <preset>`
- 新增 HTTP:
  - `POST /v1/tabs/emulate-device`
- 新增测试:
  - `atlasx/internal/tabs/device_test.go`
  - `atlasx/cmd/atlasctl/tabs_emulate_device_test.go`
  - `atlasx/internal/daemon/tab_emulate_device_test.go`
  - `atlasx/internal/daemon/tab_emulate_device_contract_test.go`

## 审查结论

- 已确认设备模拟只作用于指定 page target:
  - 主链先按 `targetID` 定位 page target，再通过该 target 的 websocket 执行 CDP 命令
- 已确认 CLI 与 HTTP 共用同一套预设和 CDP 调用逻辑:
  - 都调用 `tabs.EmulateDevice(...)`
- 已确认失败显式暴露:
  - 非法 preset 显式返回错误
  - websocket 失败显式返回错误
- 已确认 `off` 为显式清除:
  - 执行 `Emulation.clearDeviceMetricsOverride`
  - 再执行 `Emulation.setTouchEmulationEnabled(enabled=false)`
