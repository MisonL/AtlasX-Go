# CR-T096

- 日期: 2026-04-08
- 任务: `T096 补齐按建议组整理到指定窗口入口`
- 结论: 已完成。`atlasctl tabs organize-group-into-window` 与 `POST /v1/tabs/organize-group-into-window` 已落地，CLI 与 HTTP 共用 `tabgroups.ApplyGroupToWindow` 主链，先复用既有 `tabgroups.Suggest` 定位指定建议组，再复用 `windows` 与 `move-to-window` 主链把该组标签整理到指定现有窗口，并显式返回 `moved_targets` 与 `aligned_targets`。

## 变更事实

- `tabgroups.ApplyResult` 新增 `aligned_targets`
- 新增 `tabgroups.ApplyGroupToWindow(...)`
- 新增 CLI:
  - `atlasctl tabs organize-group-into-window <group-id> <window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/organize-group-into-window`
- 新增测试:
  - `atlasx/internal/tabgroups/apply_test.go`
  - `atlasx/cmd/atlasctl/tabs_organize_group_into_window_test.go`
  - `atlasx/internal/daemon/tab_organize_group_into_window_test.go`
  - `atlasx/internal/daemon/tab_organize_group_into_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有三条主链:
  - 先走 `tabgroups.Suggest(...)` 定位指定建议组
  - 再走 `Windows()` 判断组内标签是否已在目标窗口
  - 最后仅对不在目标窗口的标签走 `MoveToWindow(...)`
- 已确认 CLI 与 HTTP 共用同一套 organize into window 主链:
  - 都调用 `tabgroups.ApplyGroupToWindow(...)`
- 已确认失败显式暴露:
  - `group_id` 不存在时显式返回错误
  - `window_id` 不存在时显式返回错误
  - 任一标签迁移失败时显式返回错误
  - 不把已在目标窗口中的标签静默吞没，而是显式返回 `aligned_targets`
