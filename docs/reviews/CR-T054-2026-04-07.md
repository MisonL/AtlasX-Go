## 变更范围

- 新增 `internal/memory`，用 `Application Support/AtlasX/memory/events.jsonl` 作为 memory 事实源
- 定义两类显式事件：`page_capture` 与 `qa_turn`
- 提供 `AppendPageCapture`、`AppendQATurn`、`Load`、`LoadSummary`，不暴露通用写接口
- 扩展 `macos.Paths` 与 `/v1/status`，输出 memory root/file/present/count/last_event_at/last_event_kind
- 补充 memory 与 daemon 测试，覆盖追加、加载、空态和 bootstrap 摘要

## 审查结论

- memory v1 现在是单一文件型状态面，没有引入外部数据库或第二套事实源。
- 事件模型只允许显式 capture / qa turn 两类入口，避免提前暴露通用写接口。
- 无 memory 文件时，状态面保持 `present=false`、`count=0`、最近事件字段为空，边界显式。
