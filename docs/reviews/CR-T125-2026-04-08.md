## 实现事实

- `tabgroups` 主链新增：
  - `Inspect(client)` 与 `BuildView(windows)`
  - 只依赖既有 `Windows()` 结果，不新增第二套窗口探测逻辑
  - 继续复用既有 `Suggest(...)` 规则推导分组
  - 为每个 group 补充 `inferred`、`window_returned`、`window_ids`、`windows`
  - 明确输出这是推导视图而不是原生 Chromium tab group
- CLI 新增：
  - `tabs groups`
  - 输出 `inferred`、`returned`、`group_id`、`label`、`window_returned`、`window_id` 与 targets
- HTTP 新增：
  - `GET /v1/tabs/groups`
  - 返回结构化 JSON：`inferred`、`returned`、`groups`
  - 错误路径继续显式返回，不吞错

## 审查结论

- 新能力没有伪装成原生 tab group 控制，只暴露结构化只读推导视图
- CLI 与 HTTP 共用同一套 `tabgroups.Inspect` 主链，没有把分组推导逻辑复制到两层
- 无可分组 page target 时会显式返回 `returned=0` 和空 `groups`，不伪造分组结果
