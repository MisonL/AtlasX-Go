# CR-T093

- 日期: 2026-04-08
- 任务: `T093 补齐单标签拆到新窗口入口`
- 结论: 已完成。`atlasctl tabs move-to-new-window` 与 `POST /v1/tabs/move-to-new-window` 已落地，CLI 与 HTTP 共用 `tabs.MoveToNewWindow` 主链，先复用 `Windows()` 定位源标签所属窗口，再复用 `open-window` 与 `close(target)` 主链把单个 page target 拆到新窗口，返回结构化拆窗结果。

## 变更事实

- 新增 `tabs.WindowMoveToNewResult`
- 新增 `tabs.MoveToNewWindow(...)`
- 新增 CLI:
  - `atlasctl tabs move-to-new-window <tab-id>`
- 新增 HTTP:
  - `POST /v1/tabs/move-to-new-window`
- 新增测试:
  - `atlasx/internal/tabs/window_move_new_test.go`
  - `atlasx/cmd/atlasctl/tabs_move_to_new_window_test.go`

  - `atlasx/internal/daemon/tab_move_to_new_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有三条主链:
  - 先走 `Windows()` 定位源标签所属窗口
  - 再走 `OpenWindow(...)` 在新窗口中打开标签
  - 最后走 `Close(targetID)` 关闭源标签
- 已确认 CLI 与 HTTP 共用同一套 move to new window 主链:
  - 都调用 `tabs.MoveToNewWindow(...)`
- 已确认失败显式暴露:
  - 指定 page target 不存在时显式返回错误
  - `open-window` 失败时显式返回错误
  - `close(target)` 失败时显式返回错误
