- 新增 `atlasctl sidebar selection-ask <tab-id> <question>`
- 新增 `--selection-text` 与 `--provider-id` 参数，显式选区优先，缺省时自动抓浏览器原生选区
- 复用既有 sidebar CLI helper，统一配置校验、page context 抓取、memory 检索、runtime state 落盘和 memory 事件写入
- 补充 atlasctl 测试，覆盖显式选区成功、自动抓选区成功、provider 失败和空选区失败
- 更新 README、阶段对齐与当前状态文档，收口新的 CLI 能力事实

## 审查结论

- 本轮补齐了侧边栏选区问答在 CLI 控制面上的缺口，本地操作者不再只能通过 HTTP 调用 selection ask。
- `sidebar selection-ask` 复用了既有 `sidebar ask/summarize` 的 CLI helper，没有引入第二套 provider、runtime state 或 memory 逻辑。
- 显式 `--selection-text` 与浏览器原生选区自动抓取共存；页面无选区或 provider 失败时都会显式失败并写入 runtime state，不存在静默回退。
