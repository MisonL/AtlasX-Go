## 验证记录

1. `cd atlasx && go test ./...`
   - 退出码: 0
   - 摘要: 全量 Go 测试通过，包含 Google Chrome 风格 bundle 测试和多可执行错误测试。

2. `tmp=$(mktemp -d); src="$tmp/src/Google Chrome.app/Contents/MacOS"; mkdir -p "$src"; printf '#!/bin/sh\nexit 0\n' > "$src/Google Chrome"; chmod +x "$src/Google Chrome"; printf '<plist></plist>\n' > "$tmp/src/Google Chrome.app/Contents/Info.plist"; HOME="$tmp" go run ./cmd/atlasctl runtime stage --bundle-path "$tmp/src/Google Chrome.app" --version 123.0.0`
   - 退出码: 0
   - 摘要: stage 成功，manifest 记录的 `binary_path` 为 `.../Contents/MacOS/Google Chrome`。

3. `HOME="$tmp" go run ./cmd/atlasctl doctor`
   - 退出码: 0
   - 摘要: 输出 `chrome_source=managed_auto` 与 `chrome_binary=.../Google Chrome`。

## 审查结论

- `runtime stage` 现在通过扫描 `Contents/MacOS` 下的单一主可执行文件来识别兼容 bundle。
- 当 bundle 中没有可执行文件或存在多个可执行文件时，会显式失败，不会猜测主入口。
