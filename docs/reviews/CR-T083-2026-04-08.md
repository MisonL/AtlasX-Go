# CR-T083

- 日期: 2026-04-08
- 任务: `T083 补齐浏览器窗口状态控制入口`
- 结论: 已完成。`atlasctl tabs set-window-state` 与 `POST /v1/tabs/window-state` 已落地，CLI 与 HTTP 共用 `tabs.SetWindowState` 主链，通过 browser websocket 调用 `Browser.setWindowBounds` 对指定窗口应用显式 `windowState`，并通过 `Browser.getWindowBounds` 回读结构化窗口 bounds。

## 变更事实

- 新增 `tabs.WindowBounds`
- 新增 `tabs.SetWindowState(...)`
- 新增 CLI:
  - `atlasctl tabs set-window-state <window-id> <state>`
- 新增 HTTP:
  - `POST /v1/tabs/window-state`
- 新增测试:
  - `atlasx/internal/tabs/window_state_test.go`
  - `atlasx/cmd/atlasctl/tabs_window_state_test.go`
  - `atlasx/internal/daemon/tab_window_state_test.go`
  - `atlasx/internal/daemon/tab_window_state_contract_test.go`

## 审查结论

- 已确认动作只修改指定 `window_id` 的 `windowState`:
  - 主链调用 `Browser.setWindowBounds`，只传 `windowId` 和 `bounds.windowState`
- 已确认 CLI 与 HTTP 共用同一套窗口状态主链:
  - 都调用 `tabs.SetWindowState(...)`
- 已确认失败显式暴露:
  - 非法 `window_id` 显式返回错误
  - 非法 `state` 显式返回错误
  - browser websocket 缺失或状态设置失败显式返回错误
