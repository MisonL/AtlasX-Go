- 新增 CLI:
  - `atlasctl tabs extract-context <target-id>`
- 新增 HTTP:
  - `GET /v1/tabs/semantic-context?id=<target-id>`
- 新增测试:
  - `atlasx/internal/tabs/semantic_test.go`

## 审查结论

- 已确认入口保持只读:
  - 仅通过 `Runtime.evaluate` 提取 headings、links、forms 最小 DOM 语义摘要
  - 未调用 `memory.AppendPageCapture`、`memory.AppendQATurn` 或任何写入动作
- 已确认 CLI 与 HTTP 共用同一套逻辑:
  - 都调用 `tabs.CaptureSemanticContext(...)`
- 已确认失败显式暴露:
  - CLI 失败时打印 `capture_error`
  - HTTP 失败时返回结构化 `capture_error` 与 `error`
- 已确认空结果显式返回:
  - 无 headings、links、forms 时返回 `returned=0` 与空数组
