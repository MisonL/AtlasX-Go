## 实现事实

- `settings.Config` 新增：
  - `memory_page_visibility_enabled`
  - 兼容旧配置缺省值为 `true`
  - `settings.View` 同步暴露该字段
- `memory` 主链新增：
  - `SetPageVisibilityEnabled(paths, enabled)`
  - `UpdateControls(paths, update)`
  - `FindRelevantSnippetsForPage(paths, input)`
  - `ParsePageVisibilityValue(...)`
- `memory.LoadControls(paths)` 现在统一返回：
  - `persist_enabled`
  - `page_visibility_enabled`
- CLI 新增：
  - `atlasctl memory set-page-visibility <visible|hidden>`
  - `atlasctl memory controls` 文本输出新增 `page_visibility_enabled`
- HTTP 扩展：
  - `GET /v1/memory/controls` 返回 `page_visibility_enabled`
  - `POST /v1/memory/controls` 现在支持独立更新 `persist_enabled`、`page_visibility_enabled` 或同时更新两者
- page-scoped retrieval 接入：
  - `tabs memories`
  - `tabs suggest`
  - `tabs recommend-context`
  - `tabs agent-plan`
  - `sidebar ask`
  - `sidebar summarize`
  - `sidebar selection-ask`
  - 当开关关闭时，上述动作仍执行，但 memory snippets 返回空
- 非 page-scoped retrieval 保持不变：
  - `atlasctl memory search`
  - `GET /v1/memory/search`

## 审查结论

- 页面级 memory 注入已经被单一 helper 收口，没有在 CLI/daemon 入口散落判断
- 控制开关只影响 page-scoped retrieval，不影响 `memory list/search` 与 `memory/events.jsonl` 写路径
- sidebar 在页面可见性关闭时仍可成功调用 provider，但不会把旧 memory snippets 注入请求
