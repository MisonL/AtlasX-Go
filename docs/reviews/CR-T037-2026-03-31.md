- 任务: T037 设计 managed runtime 下载器状态机骨架
- 日期: 2026-03-31
- 结论: 通过

## 变更范围

- 新增 `InstallPlan` 与 `InstallPhase`
- 新增 `NewInstallPlan` 和 `AdvanceInstallPlan`
- 明确 URL 校验、失败态和回滚态，但不触发真实下载

## 状态转换

- `planned -> downloading`
  - 事件: `start_download`
  - 约束: 仅允许从 `planned` 进入

- `downloading -> downloaded`
  - 事件: `finish_download`

- `downloaded -> verifying`
  - 事件: `start_verify`

- `verifying -> verified`
  - 事件: `finish_verify`

- `verified -> staging`
  - 事件: `start_stage`

- `staging -> staged`
  - 事件: `finish_stage`

- `downloading/downloaded/verifying/verified/staging/failed -> rollback`
  - 事件: `start_rollback`

- `rollback -> rolled_back`
  - 事件: `finish_rollback`

- `非终态 -> failed`
  - 事件: `fail`
  - 约束: 必须携带失败原因，且 `staged/rolled_back` 不允许回退成 `failed`

## 审查结论

- 当前实现只定义下载控制面的状态机，不执行真实网络下载，也不伪造“已安装成功”。
- 后续若接入真实下载器，只需要把下载、校验、stage、rollback 具体执行器挂接到这套状态转换即可。
