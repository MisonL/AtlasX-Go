## 实现事实

- 配置新增 `memory_hidden_hosts`
- `memory controls` 响应新增 `hidden_hosts`
- CLI 新增：
  - `atlasctl memory set-site-visibility <host> <visible|hidden>`
- 页面级 retrieval 现在按以下顺序生效：
  - 先检查全局 `page_visibility_enabled`
  - 再检查当前页面 host 是否被隐藏
  - 再过滤掉所有来自隐藏 host 的 memory 事件
- `memory list/search` 与 memory 事件持久化主链不受按站点隐藏规则影响

## 审查结论

- 按站点隐藏规则只作用于页面级 snippets 注入，没有修改 memory 事件写入、memory list 或直接 memory search 行为
- `atlasctl` 与 `atlasd` 共用同一套 `memory.ControlsUpdate` 与 `FindRelevantSnippetsForPage` 主链，没有引入第二套过滤逻辑
- host 输入会显式规范化；非法 host 输入显式失败，不静默跳过
