## 实现事实

- `permissions.Status` 当前统一导出以下代码边界事实:
  - `source`
  - `native_bridge_present`
  - `granted_state_observable`
  - `accessibility_probe_supported`
  - `screen_recording_probe_supported`
  - `automation_probe_supported`
  - `full_disk_access_probe_supported`
  - `permission_prompt_supported`
  - `permission_write_supported`
  - `permission_state_persisted`
  - `os_policy_failures_surface`
  - `notes`
- 当前固定事实:
  - 尚未实现任何真实 macOS TCC 权限探测
  - 尚未实现权限提示或授权写路径
  - 尚未持久化权限状态
  - OS 权限拒绝预期会按原始错误显式暴露
- 明确不做的内容:
  - 不伪造系统已授权或未授权
  - 不引入 AppleScript/TCC 原生桥接写路径
  - 不把“未实现探测”误报成“权限正常”

## 审查结论

- CLI 与 HTTP 共用同一套 permissions 状态真相，没有分叉实现
- 输出严格限制在代码事实边界，没有伪造任何真实 macOS 授权状态
- 当前 OS 权限相关失败仍应以运行时错误排障，不应把本接口误用为 TCC 已授权证明
