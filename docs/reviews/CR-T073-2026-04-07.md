## 变更范围

- 新增 `atlasctl memory list`，支持 `--limit` 只查看最近 N 条 memory events
- 新增 `GET /v1/memory`，结构化返回 memory root、events file、事件摘要与最近事件列表
- `internal/memory` 新增最近事件读取能力，继续以 `memory/events.jsonl` 为单一事实源
- 补充 CLI、daemon 与 memory 测试，覆盖空态、limit 与契约字段
- 更新 README、阶段对齐与当前状态文档，收口新的 memory 读入口事实

## 审查结论

- 本轮补齐了 Browser memories 的最小读控制面，操作者现在可以直接查看本地 memory 事件，而不必手工翻 `events.jsonl`。
- 新入口严格只读，没有新增写路径，也没有引入第二套 memory 状态或索引实现。
- 无 memory 文件时返回空列表和显式摘要，`limit` 非法时显式失败，符合 Debug-First 约束。
