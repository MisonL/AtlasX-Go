# CR-T086

- 日期: 2026-04-08
- 任务: `T086 补齐浏览器窗口关闭入口`
- 结论: 已完成。`atlasctl tabs close-window` 与 `POST /v1/tabs/close-window` 已落地，CLI 与 HTTP 共用 `tabs.CloseWindow` 主链，先复用 `windows` 分组结果定位目标窗口，再逐个关闭窗口中的 page targets，返回结构化关闭结果。

## 变更事实

- 新增 `tabs.WindowCloseResult`
- 新增 `tabs.CloseWindow(...)`
- 新增 CLI:
  - `atlasctl tabs close-window <window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/close-window`
- 新增测试:
  - `atlasx/internal/tabs/window_close_test.go`
  - `atlasx/cmd/atlasctl/tabs_close_window_test.go`
  - `atlasx/internal/daemon/tab_close_window_test.go`
  - `atlasx/internal/daemon/tab_close_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有两条主链:
  - 先走 `Windows()` 定位 `window_id`
  - 再逐个走 `Close(targetID)` 关闭窗口中的 page targets
- 已确认 CLI 与 HTTP 共用同一套 close window 主链:
  - 都调用 `tabs.CloseWindow(...)`
- 已确认失败显式暴露:
  - `window_id` 不存在时显式返回错误
  - 任一 `target` 关闭失败时显式返回错误
