# CR-T102

- 日期: 2026-04-08
- 任务: `T102 补齐按指定窗口单建议组整理到指定现有窗口入口`
- 结论: 已完成。`atlasctl tabs organize-window-group-into-window` 与 `POST /v1/tabs/organize-window-group-into-window` 已落地，CLI 与 HTTP 共用 `tabgroups.ApplyWindowGroupToWindow` 主链，先基于既有窗口分组定位指定源窗口，再仅对该窗口中的 page targets 生成建议组并定位指定 `group_id`，最后复用 `applyGroupToWindow` 主链把该组整理到指定现有目标窗口。

## 变更事实

- 新增 `tabgroups.ApplyWindowGroupToWindow(...)`
- 新增窗口内单组定向整理结果:
  - `ApplyWindowGroupIntoWindowResult`
- 新增 CLI:
  - `atlasctl tabs organize-window-group-into-window <source-window-id> <group-id> <target-window-id>`
- 新增 HTTP:
  - `POST /v1/tabs/organize-window-group-into-window`
- 新增测试:
  - `atlasx/internal/tabgroups/apply_test.go`
  - `atlasx/cmd/atlasctl/tabs_organize_window_group_into_window_test.go`
  - `atlasx/internal/daemon/tab_organize_window_group_into_window_test.go`
  - `atlasx/internal/daemon/tab_organize_window_group_into_window_contract_test.go`

## 审查结论

- 已确认入口只复用现有两层主链:
  - 先走 `Windows()` 定位指定源窗口中的 page targets 并生成建议组
  - 再对指定组走 `applyGroupToWindow(...)`
- 已确认 CLI 与 HTTP 共用同一套窗口内单组定向整理主链:
  - 都调用 `tabgroups.ApplyWindowGroupToWindow(...)`
- 已确认失败显式暴露:
  - `source_window_id` 非法时显式返回错误
  - `target_window_id` 非法时显式返回错误
  - `group_id` 缺失或不存在时显式返回错误
  - 源窗口不存在时显式返回错误
  - 目标窗口不存在时显式返回错误
  - `source_window_id == target_window_id` 时显式返回错误
  - 任一标签迁移失败时显式返回错误
